Apache Hardening

Configuration sécurisée d'un serveur Apache avec SSL, durcissement et bonnes pratiques de sécurité web.

Contexte

Sécurisation complète d'un serveur web Apache hébergeant une application sensible. Application des recommandations CIS (Center for Internet Security) et OWASP.

Stack Technique

  • Linux (Ubuntu Server)
  • Apache HTTP Server 2.4
  • OpenSSL (TLS 1.3)
  • ModSecurity (WAF)
  • Fail2Ban

Points Clés

  • TLS : Configuration TLS 1.3 avec cipher suites restreintes
  • WAF : ModSecurity avec règles OWASP Core Rule Set
  • Protection : Fail2Ban pour prévention des attaques par brute-force
  • Headers : Security Headers (CSP, HSTS, X-Frame-Options, etc.)
  • Audit : Score A+ sur SSL Labs après durcissement

À propos de la cybersécurité

Ce domaine regroupe mes travaux en cybersécurité offensive et défensive, incluant le pentesting, le hardening de systèmes Linux et web, ainsi que des environnements de test sécurisés pour l'analyse de comportements systèmes.

J'ai développé une expertise dans :

  • Le hardening de systèmes (Linux, Apache, Docker) selon les benchmarks CIS
  • Le pentesting avec Kali Linux et les outils standards
  • L'analyse de vulnérabilités avec OpenVAS, Nikto, Nmap
  • La configuration sécurisée des services web et cloud
  • La création de labs sécurisés pour des tests sans risque
